Cavoke features

Cross-platform

Cavoke, thanks to it design, is cross-platform, meaning that any device may play the games.

Easy to use

Python library is well documented and provided with handful examples. The library itself was built to be able to be easily built on.

Servers

Cavoke is being run on special servers, that do all the hard work for your game session.

Price

Free. completely. forever.

Open-source

Cavoke is open-source. You can find the source code on GitHub.

Maintainability

Project is being well-maintained and every issue or pull request will be review in the matter of hours.

Structure

How it works

This project consists of three separate sub-projects:

  • Python library (cavoke-lib)
  • Server (cavoke-server)
  • Client-side (cavoke-web, more coming soon-ish)

By splitting up server- and client-side, we can achieve being a cross-platform project, as with simple API games can be accessed from any device. Here you can see the approximate scheme of the project:

Scheme

Now we will try to explain every part on this scheme.


Python library and developer's code

The games are not going to write themselves. If you'd like to create your game, you will have to write your masterpiece using Python programming language and our library to make the coding and deploying much easier.

Our python library is called cavoke. It can be downloaded with pip package manager: pip install cavoke. Alternatively, it can be cloned from GitHub.

This library implements classes like Game, Image, Grid, Row, etc. They will help you write the model of your game. Due to project-design, library doesn't have the rendering capabilities and returns only JSON field representation to be rendered at client-side.

See the documentation and examples. If you have any questions regarding cavoke-lib and cavoke in general feel free to reach out to us.

Git repository

After your code is done, you should publish it (or no one will ever see it). To do so you should publish the git repository containing your code and provide us with the link to your git file (we recommend using GitHub for this part).

With this link you can start publsihing your work.

Server-executor and database

Server-executor is a simple Django server, that listens to incoming API calls and accesses games directly. Database is a small NoSQL database, used for storing game info and other stuff.

Developer's git repsository will be downloaded and the code will be run. Obviously, this poses a great security threat. Unfortunately, there is no way of dealing with it except for manual moderation, that's exactly what we do.

Client-side

Finally, there is the client-side. You can use cavoke from web, phone and desktop (some options might be unavailable). The client-side application receives JSON from the server and renders it on the user's screen.
